## 🗺️ El Plan de Estudios del Bootcamp de 101 Días
Un mapa de ruta ágil, guiado por hitos, que alterna entre la ciencia teórica rigurosa de la IA y las implementaciones prácticas directamente en la terminal.

### 🧠 Módulo 00: IA y LLMs — Las Fundaciones del "Cerebro"
*Enfoque: Mecánica de inferencia, restricciones estructurales de los modelos y transición de una mentalidad orientada a píxeles hacia la intención lógica pura.*

- [x] **Día 01: El Panorama Global (AI for Everyone)** — Límites operacionales de los LLM, mapeo de lo que la inteligencia artificial no es capaz de ejecutar e impactos prácticos en el mercado tecnológico global.
- [x] **Día 02: La Matemática de la Elección** — Mecánica de la regresión lineal, calibración de pesos ($Weights$) y optimización de sesgos ($Biases$) para la relevancia algorítmica de los datos.
- [x] **Día 03: Deep Learning y Redes Neuronales** — Arquitecturas de capas profundas interpretadas como funciones encadenadas complejas (donde la salida de una capa alimenta la entrada de la siguiente).
- [x] **Día 04: El Punto de Inflexión: Transformers y Modelos de Razonamiento** — Estudio profundo del mecanismo de Self-Attention (Autoatención) y procesamiento de ventanas de contexto masivas. Introducción a los nuevos paradigmas de modelos de razonamiento nativo entrenados mediante Aprendizaje por Refuerzo (Reinforcement Learning).
- [x] **Día 05: Tokenización y Embeddings** — Vectorización práctica de datos textuales y análisis de la transformación estructural de strings a tensores numéricos mediante `tiktoken`.
- [x] **Día 06: Ecosistema Hugging Face y Pesos Locales** — Navegación en el hub de código abierto, selección de arquitecturas de modelos (Llama-3/Mistral) y configuración de Ollama para la ejecución local sin conexión en Ubuntu Linux.
- [x] **Día 07: Pipelines de Inferencia en Código** — Construcción de scripts iniciales en Python para cargar pesos de modelos y ejecutar flujos de inferencia en localhost.
- [x] **Día 08: MLOps y Gestión del Ciclo de Vida** — Gestión de la degradación de modelos, control de versiones de parámetros y analogía estructural con el mantenimiento de núcleos de WordPress en producción.
- [x] **Día 09: Contenedorización de IA con Docker** — Empaquetamiento de endpoints de modelos en contenedores ligeros para el consumo de APIs estructuradas por parte de aplicaciones externas y móviles.
- [x] **Día 10: Ética, Viabilidad y Plan de Vuelo** — Revisión analítica, gobernanza de datos e integración inicial de herramientas bajo las operaciones de Personal Help Desk.

### 🔌 Módulo 01: APIs y Prompt Engineering — Versión de Élite
*Enfoque: Frameworks industriales de orquestación de prompts y aseguramiento de salidas predecibles de los modelos mediante validación de código en tiempo de ejecución.*

- [x] **Día 11: Tokenización Avanzada e Internas de LLMs** — Procesamiento autorregresivo, distribución de probabilidad Softmax, optimización de KV Cache en la ventana de contexto y el impacto del algoritmo BPE (Byte Pair Encoding) en idiomas latinos.
- [x] **Día 12: Anatomía de un Prompt Eficaz y Ingeniería de Contexto** — Implementación multinivel del Framework RTOS (Role, Task, Context, Constraint), etiquetas estructurales (delimitadores XML/Markdown) y mitigación activa del sesgo de pocas muestras (Few-shot bias).
- [x] **Día 13: Razonamiento Lógico y Descomposición de Contexto** — Reducción estructural de alucinaciones mediante paradigmas avanzados de prompting: Chain of Thought (CoT), Self-Consistency y Least-to-Most prompting.
- [x] **Día 14: Hiperparámetros Técnicos y Salidas Estructuradas** — Control fino sobre los parámetros de muestreo del modelo (Temperatura, Top-P, Penalizaciones de Frecuencia/Presencia) y cumplimiento rígido de esquemas JSON utilizando la biblioteca `Pydantic` en Python.
- [x] **Día 15: Frameworks Avanzados (RAG, Vectores y Tool Use)** — Flujos de recuperación de datos externos, generación de embeddings independientes y mecánica sistémica de llamadas a funciones por parte de los LLM (Function Calling / Tool Use).
- [x] **Día 16: MLOps Defensivo, Seguridad y Auditoría** — Blindaje de sistemas contra Prompt Injection y Jailbreaking; ejecución de validaciones a escala basadas en patrones de LLM-as-a-judge y monitoreo de logs de producción en tiempo de ejecución.

### 📐 Módulo 02: Matemática para Vectores y Embeddings — Un Enfoque Visual
*Enfoque: Transposición de la intuición espacial 3D y vectorial de los softwares gráficos (Figma/Illustrator) hacia la geometría multidimensional de la IA.*

- [x] **Día 17: El Espacio Conceptual** — Definición geométrica de un embedding como una firma de identidad de alta dimensión; dominio de la Proximidad Semántica a través de espacios latentes densos (768 a 1536 dimensiones).
- [x] **Día 18: Midiendo Relaciones** — Aplicación mecánica de la Similitud de Coseno para calcular matemáticamente ángulos ideológicos:
    $$\cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$$
    Comprensión del Producto Escalar (Dot Product) como el motor central que impulsa el Mecanismo de Atención en los Transformers, y visualización práctica de comportamientos vectoriales en RAG utilizando herramientas como el TensorFlow Projector.

### 🌐 Módulo 03: Model Context Protocol (MCP) — Arquitectura y Hosts Nativos
*Enfoque: Romper silos de información mediante el diseño, depuración y orquestación de servidores de protocolo unificado en Ubuntu Linux.*

- [x] **Día 19: Concepto Central y Arquitectura del Protocolo** — Mapeo detallado del ciclo de vida Host ↔ Server ↔ Client, analizando sus ventajas estructurales frente al Function Calling tradicional en APIs aisladas.
- [x] **Día 20: Exploración del Ecosistema** — Configuración de instancias de control nativas mediante `claude_desktop_config.json`, integración de conectores de código abierto (SQLite, GitHub, Google Maps) y sandbox seguro para archivos locales.
- [x] **Día 21: Construcción de Servidores MCP Personalizados** — Desarrollo de extremo a extremo utilizando los SDK oficiales de Python y TypeScript, mapeando Resources (lectura), Tools (acciones/escritura) y Prompts dinámicos (plantillas de contexto).
- [x]  **Día 22: Integración Avanzada, Depuración y Producción** — Blindaje de la comunicación sobre transportes Stdio y HTTP, gestión de limitaciones de tokens de contexto y resolución de bloqueos mutuos (deadlocks) en buses de E/S asíncronos.

### 🧭 Extensión Especial: Infraestructura de Node, NPM, NPX y Python
*Núcleo Teórico: Profundización en el protocolo JSON-RPC sobre transportes Stdio (stdin/stdout). Diagnóstico de corrupción de flujos de datos causada por salidas no estructuradas en segundo plano (como el uso indebido de comandos print() globales).*

- [x] **Paso 1: Auditoría de Contratos Públicos (Filesystem Hub)** — Inspección del contrato oficial JSON-RPC de Filesystem de Anthropic a través del MCP Inspector, entendiendo cómo el agente reclama capacidades seguras de lectura y escritura delimitadas por rutas absolutas en Linux.
- [x] **Paso 2: Configuración de Hosts Nativos (Claude Desktop Integration)** — Transición fuera de los entornos de prueba hacia la conexión de servidores propios (Python) y públicos (Node.js) directamente al "corazón" de entornos AI Host reales mediante configuraciones globales.
- [x] **Paso 3: Orquestación Multi-Servidor y Concurrencia de E/S** — Análisis del comportamiento del Host bajo flujos concurrentes de entrada/salida, garantizando que el agente consulte múltiples servidores simultáneamente sin bloquear la memoria RAM o generar conflictos en el bus de datos.
- [x] **Paso 4: Auditoría de Logs de Producción y Seguridad Avanzada** — Forzado de errores de conexión reales (p. ej., Broken Pipe, Permiso Denegado) y utilización de comandos de terminal (`tail -f`) para depurar logs de streaming en bruto en tiempo real.

### 🗄️ Módulo 05: Arquitectura de Sistemas RAG Profesionales
*Enfoque: Ingestión avanzada, extracción, modelado vectorial y recuperación determinista de datos corporativos no estructurados.*

- [ ] **Día 34: Pipelines de Ingestión: RAG vs. Fine-Tuning** — Evaluación arquitectónica de la recuperación de contexto externo dinámico frente a la incrustación de conocimiento especializado directamente en la memoria paramétrica del modelo.
- [ ] **Día 35: Carga de Documentos y Parseo de Alta Fidelidad** — Extracción y limpieza de texto sin procesar proveniente de PDFs complejos de múltiples columnas, documentación en Markdown y cargas útiles de CMS personalizados (HTML/WordPress).
- [ ] **Día 36: Estrategias Avanzadas de Chunking** — Implementación de fragmentación estructural, basada en tokens y semántica para preservar los contextos primarios de los fragmentos padres.
- [ ] **Día 37: Evaluación de Modelos de Embedding** — Ejecución de benchmarks entre APIs cerradas propietarias y pesos de embedding de código abierto ejecutados localmente mediante Hugging Face.
- [ ] **Día 38: Bases de Datos Vectoriales I** — Configuración práctica, escalado e indexación de coordenadas matemáticas vectoriales en almacenes líderes de la industria (ChromaDB, Pinecone o Weaviate).
- [ ] **Día 39: Mecánica de Indexación por Similitud** — Configuración de algoritmos de indexación adaptados para la Similitud de Coseno frente al Producto Escalar o la Distancia Euclidia.
- [ ] **Día 40: Pipeline de Línea Base de Extremo a Extremo** — Construcción de un pipeline automatizado que ingesta un activo bruto, parsee el contenido, genere embeddings y persista los índices vectoriales localmente.
- [ ] **Día 41: Frameworks de Recuperación Avanzada** — Mitigación de la pérdida de información mediante la implementación de sistemas de Parent Document Retrieval (Recuperación de Documentos Padre) y Sentence Window Retrieval.
- [ ] **Día 42: Sistemas de Transformación de Consultas** — Automatización del refinamiento de consultas a través de la generación algorítmica de múltiples consultas y Embeddings de Documentos Hipotéticos (HyDE).
- [ ] **Día 43: Recuperación en Dos Etapas con Modelos de Reranking** — Integración de capas de reordenamiento neuronal (Cohere Rerank o BGE Reranker) para maximizar la relevancia de los top-$K$ contextos.
- [ ] **Día 44: Context Stuffing y Citación de Fuentes** — Formateo defensivo de los documentos recuperados dentro de los prompts del sistema, forzando al LLM a generar citaciones rígidas y sin alucinaciones de los textos de origen.
- [ ] **Día 45: Evaluación Sistémica con RAGAS** — Cuantificación de métricas de calidad del pipeline a través del framework Ragas, midiendo la Fidelidad (Faithfulness), Relevancia de la Respuesta y Recuperación del Contexto.
- [ ] **Día 46: Gráficos de Conocimiento y GraphRAG** — Extracción de entidades estructurales y relaciones para construir Gráficos de Conocimiento multidocumento capaces de responder preguntas globales y macro de un ecosistema técnico.
- [ ] **Día 47: Proyecto Capitular del Módulo** — Construcción de una solución de búsqueda semántica para actuar como base de conocimiento avanzada sobre un portafolio técnico diversificado, reemplazando la coincidencia de strings tradicional.

### 📊 Módulo 07: Ingeniería de Datos y Fine-Tuning

- [ ] **Día 60: Curaduría e Ingeniería de Datasets** — Establecimiento de estándares industriales estrictos para el equilibrio de datos de entrenamiento, diversidad estructural y reducción de sesgos sistémicos.

- [ ] **Día 68: Fine-Tuning de LLMs para Prosa Especializada** — Ejecución de bucles de entrenamiento personalizados en modelos abiertos ligeros (Llama-3-8B / Gemma-2) para restricciones de generación de texto específicas.
- [ ] **Día 69: Fine-Tuning Especializado en Código** — Entrenamiento de un modelo local para que coincida con un estilo de diseño de código preciso y personalizado, adhiriéndose estrictamente a frameworks CSS especializados y patrones PHP.
- [ ] **Día 70: Telemetría y Optimización del Entrenamiento** — Seguimiento de curvas de pérdida (Loss Curves) mediante configuraciones de validación, identificando el Sobreajuste (Overfitting) de forma temprana y mitigando el Olvido Catastrófico (Catastrophic Forgetting).
- [ ] **Día 71: Fine-Tuning Multimodal y Adaptación Visual** — Introducción a la especialización de Modelos de Lenguaje y Visión (VLMs) y configuración de parámetros de entrenamiento visual (como LoRAs personalizados para Stable Diffusion).
- [ ] **Día 72: Alineación de Modelos mediante DPO** — Implementación de la Optimización de Preferencia Directa (DPO) para alinear los comportamientos del modelo directamente con conjuntos de datos de preferencia humana explícita.
- [ ] **Día 73: Benchmarking de Pesos Personalizados** — Diseño de pruebas comparativas a ciegas para evaluar modelos especializados frente a líneas base públicas.
- [ ] **Día 74: Exportación y Cuantización de Artefactos de Modelos** — Conversión de pesos de modelos sin procesar en formatos de distribución optimizados (GGUF / Safetensors) para una inferencia local rápida.
- [ ] **Día 75: Proyecto Capitular Consolidador del Módulo** — Compilación de un dataset propietario, lanzamiento de un pipeline de entrenamiento de extremo a extremo y servicio del modelo recién personalizado de forma local.
